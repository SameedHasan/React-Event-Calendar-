# react-event-calendar-suite

Interactive demos for the calendar component. Use the **Calendar** stories to progressively explore features.

## Install

```bash
npm install react-event-calendar-suite
```

Peer dependencies:

```bash
npm install react react-dom antd @ant-design/icons dayjs zustand
```

## Minimal setup

The calendar is a **display component by default**. Pass only `events` and it works:

```jsx
import Calendar from 'react-event-calendar-suite';
import 'react-event-calendar-suite/style.css';

export function App() {
  return (
    <Calendar
      events={events}
      defaultView="month"
    />
  );
}
```

## Opt-in features

Features are disabled until you explicitly enable them:

| Feature | How to enable |
| :--- | :--- |
| Add / edit / delete events | Pass `onAddEvent`, `onUpdateEvent`, `onDeleteEvent` + `showAddEventButton` |
| ICS export button | `showExportButton={true}` |
| Drag-and-drop move | `onEventDrop` handler |
| Event extend / resize | `onEventResize` handler |
| Date-click to create | `allowDateClick={true}` |

## Story index

| Story | What it shows |
| :--- | :--- |
| **Default** | Minimal display — `readOnly`, no CRUD, no export |
| **Interactive** | Full CRUD: add, edit, delete, export |
| **All Views** | Month / week / day / list / year |
| **Controlled** | `currentDate` + `view` owned by parent state |
| **Read Only** | `readOnly` + `onEventClick` for custom detail panels |
| **Theming** | `primaryColor`, `theme="dark"`, CSS variables |
| **Custom Render** | `renderEvent` slot override |
| **Drag And Drop** | Opt-in move + resize (`onEventDrop`, `onEventResize`) |
| **Timezone** | `timezone` prop — display events in any IANA zone |
| **Two Calendars** | Two isolated `<Calendar />` instances on one page |
